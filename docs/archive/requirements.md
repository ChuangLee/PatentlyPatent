# patent_king 需求文档

> 版本 v0.1 · 2026-05-07 · 基于调研汇总（ai_docs/research_summary.md）与 4 路调研

## 1. 项目目标

开发"专利全流程辅助系统 patent_king"，使专利代理师/发明人在 CNIPA 实务下达到资深代理师水平的产出质量。

- 形态：**Python CLI 主体 + Claude Skill 薄包装**（双壳，二者皆可独立使用）
- 优先体系：**CNIPA**（兼顾 CN/US/PCT 数据源）
- 覆盖环节：**检索 / 挖掘 / 撰写 / OA 答辩** 全流程
- 检索方案：**关键字检索各专利网站**（不做向量/embedding）

## 2. 用户与场景

| 角色 | 主要场景 |
|---|---|
| 专利代理师 | 检索 prior art、起草权要+说明书、答辩 OA |
| 企业 IP 工程师 | 挖掘交底书、规划 portfolio |
| 发明人 | 自助生成可读的交底书与初稿 |

## 3. MVP 范围（v0.1）

**目标闭环：交底→检索→撰写**

| 模块 | 功能 | 验收 |
|---|---|---|
| `pk search` | 关键字检索 CNIPA / Google Patents（关键字模式）/ EPO OPS / USPTO ODP，输出特征对照表 | 给定交底要点，返回 ≥10 篇相关现有技术 + IPC/CPC + 摘要 + 与本申请要素对照 |
| `pk draft` | 基于检索结果与交底，生成独权（含概括度等级）+ 从权梯度（≥5 项）+ 说明书五段骨架 + ≥1 实施例 | 输出符合 CN 实务格式；自动通过 R20.2 最小集 lint 与 A26.4 支持 lint |
| `pk check` | 对草稿做 A22/A26.3/A26.4/A33/R20.2/单一性 体检报告 | 输出 markdown 体检报告，标 risk 等级 |

**非 MVP（v0.2+）**：交底问题树交互、OA 答辩、portfolio 布局、附图生成

## 4. 功能需求

### 4.1 检索 (pk search)
- F-S-1 输入：自然语言交底 / 现有草稿 / 关键词
- F-S-2 自动要素拆解（领域/问题/手段/效果），输出 4 维关键词 ≥3 个/维
- F-S-3 关键词扩展：同义词、上位/下位、中英对照
- F-S-4 IPC/CPC 自动推荐 ≥3 个候选分类号
- F-S-5 多源适配器：CNIPA epub、Google Patents 关键字搜索、EPO OPS REST、USPTO ODP REST
- F-S-6 结果去重（按 family）、合并、排序（命中度+引用数+时间）
- F-S-7 输出特征对照表（markdown），逐文献 vs 本申请按"问题/手段/效果"3 维对齐
- F-S-8 X 类（破新颖性）/Y 类（破创造性）自动标注（LLM 判断 + 人工复核）

### 4.2 撰写 (pk draft)
- F-D-1 接收检索结果与交底，生成独权（强/中/弱三档概括度供选）
- F-D-2 从权梯度生成：进一步限定→可选特征→数值范围→优选实施例→解释性从权
- F-D-3 说明书五段骨架：技术领域 / 背景 / 发明内容 / 附图说明 / 具体实施方式
- F-D-4 实施例参数表 + 范围端点 + 优选值
- F-D-5 附图标号一致性（草稿内部）
- F-D-6 三段式 / 非三段式自动判别（机械结构默认三段式）

### 4.3 体检 (pk check)
- F-C-1 R20.2 必要技术特征最小集：识别独权中可删特征
- F-C-2 A26.3 充分公开：每个独权特征是否在说明书有实施例支撑
- F-C-3 A26.4 支持：功能性限定是否有结构对应、权要范围是否落在实施例覆盖
- F-C-4 A33 修改超范围（适用于改稿场景）：与"原申请文本"做特征级 diff，定位"中间概括"风险
- F-C-5 单一性 R34：多独权间是否有"特定技术特征"
- F-C-6 输出 risk 报告：高/中/低 + 修改建议

### 4.4 OA 答辩 (pk oa, v0.2)
- F-O-1 OA 通知书结构化解析（驳回理由/对比文件/审查员观点）
- F-O-2 三步法答辩链：D1 重定义 → 实际解决问题重定义 → 技术启示反驳
- F-O-3 修改方案 + A33 原文锚定校验
- F-O-4 答辩话术草稿

## 5. 非功能需求

- N-1 离线可用：除调用各专利网站和 LLM 外不依赖云组件
- N-2 LLM 可插拔：默认 Anthropic Claude（claude-opus-4-7 / claude-sonnet-4-6），可切 Qwen3 / GLM-5 等
- N-3 提示词版本化：prompts/ 下按模块组织、git 跟踪
- N-4 网站适配器抗腐烂：每个网站独立模块、失败可降级、关键 selector/接口集中配置
- N-5 输出可追溯：每个生成结论附"出处链"（检索命中/法条/规则）
- N-6 速度：单条独权生成 ≤ 60s（不含检索），检索单次 ≤ 30s

## 6. 边界（必须人审，禁止 LLM 单独决策）

1. 独权概括度终判（写多上位）
2. 最接近现有技术 D1 的重定义
3. A33 修改超范围的最终边界判断
4. portfolio 战略层（哪些申请、哪些国家）
5. 答辩 vs 分案 vs 主动放弃 的策略选择

CLI 在涉及上述决策处必须输出"候选方案 + 风险标注"，不直接定稿。

## 7. 数据源接入策略（关键字模式）

| 源 | 模式 | 优先级 | 备注 |
|---|---|---|---|
| CNIPA epub.cnipa.gov.cn | HTML 关键字搜索抓取 | P0 | CN 一手；处理验证码/限流，限速友好爬取 |
| Google Patents | 关键字 URL（`q=...`）抓取 | P0 | 多语种、含 CN 译文、免登录 |
| EPO OPS | REST + CQL 关键字 | P1 | 免费 4GB/周 |
| USPTO ODP | REST 关键字 | P1 | 免费、需 API key |
| Lens.org | REST 关键字 | P2 | 学术-专利交叉 |

**不实现**：向量检索、自建 embedding、PaECTER-zh 微调（已确认）

## 8. 验收标准（MVP）

1. 端到端：给定一份"自然语言交底（200-500 字）"，10 分钟内得到：检索报告 + 独权 3 档 + 从权 5 项 + 说明书五段骨架 + 体检报告
2. 法条 lint 命中率 ≥80%（用 5 个真实样例对照）
3. 检索召回 ≥10 篇相关；至少 1 篇真正最接近现有技术（人工评）
4. CLI 与 Skill 两种入口都能跑通

## 9. 里程碑

- M1（本周）：项目骨架 + 检索 P0 适配器（CNIPA + Google Patents） + 检索 CLI 跑通
- M2：撰写模块 + 体检 lint
- M3：Skill 薄包装 + 端到端 demo
- M4（v0.2）：OA 答辩 + 挖掘问题树
