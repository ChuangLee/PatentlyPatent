# 法条 lint 规则速查（基于审查指南 2023）

## 撰写阶段
| 规则 | 关键判断 | 雷区 |
|---|---|---|
| R20.2 必要技术特征 | 独权写"解决问题所必需的全部特征"，不多不少（最小可工作集） | 漏写=驳回，多写=保护范围窄 |
| A26.3 充分公开 | 每个权要特征在说明书须有实施例，使本领域技术人员"无需创造性劳动即可实施" | 功能性限定无实施例、数值范围只给端点 |
| A26.4 支持 | 权要范围须落在实施例覆盖内 | 上位概括过激、功能性限定无结构对应 |
| R34 单一性 | 多独权间须有相同/相应的"特定技术特征" | 缺少 → 强制分案 |

## OA 答辩阶段
| 规则 | 三步法/要点 |
|---|---|
| A22.3 创造性 | ① 抢"最接近现有技术"D1 定义权 ② 重新认定实际解决的技术问题（禁事后诸葛亮）③ 打技术启示（D1+D2 motivation？反向教导？公知常识须举证） |
| A22.2 新颖性 | 单篇对比；上下位（下位破上位，上位不破下位）；惯用手段直接置换 |
| A33 修改超范围 | "原说明书+权要书记载的或可直接毫无疑义确定" — 只做删/缩，慎做加/改；"中间概括"几乎必死 |

## 必须人审
1. 独权概括度终判
2. D1 重定义
3. A33 边界
4. portfolio 战略
5. 答辩 vs 分案 vs 放弃
