# CLAUDE.md — reasoning-posttrain 项目约定

## 项目主题

开发一个专利调研 挖掘 编写 的skill,在各个方面达到人类专家的水平


## 落地优先

注重快速落地应用。输出形态不绑定 skills，可以是可执行程序、系统、CLI、服务等任何能跑起来解决问题的形式。先做 MVP 跑通闭环，再迭代。

## 工作流 (铁律)

```
需求分析 → 调研 → 需求文档 → 计划 → 设计 → 开发 → 部署 → 验证
```

- 调研在每一个环节都可以重复使用，涉及到需求/计划/设计等都需要先更新 docs 下的文档和图，鼓励多画图
- 每一次对话都要检查是否需要更新需求/计划文档及后续文档
- **调研阶段是必经步骤 (铁律中的铁律)**：做任何事之前必须先调研有无现成工具/库/方案/开源项目
- 调研输出: 写分析等文档到 `ai_docs/` 再动手
- 调研的重点知识材料下载到 `refs/专利专家知识库/` 文件夹，作为长期知识库（系列内容 ≥3 篇必须建子文件夹，每个子文件夹含 `_目录.md`，主索引 `INDEX.md` 全中文）；优秀重点参考的项目下载到 `refs/3rd_repos/`


## 调研风格

发散 → 深入 → 继续深入(数学) → 全盘洞察 → 灵光一闪 → 系统融合 → 小心求证


## 研究优先级

1. 优先近半年内资料
2. 超过 1 年默认参考价值低 (除非奠基经典/仍被高频引用)
3. 输出显式区分：近半年高优 / 经典较老 / 可能过时
